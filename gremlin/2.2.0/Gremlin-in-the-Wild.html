This section provides an overview of the groups using Gremlin. If you use Gremlin and you are not on this list, please add yourself. If you don’t have a GitHub account, then email your usecase to us and we can add it.

  • AT&Ti: Using Gremlin to manage their graph database backend and to evaluate recommendation algorithms. Other TinkerPop packages that are used are Blueprints, Pipes, and Rexster.
  • Neo4j: Embedding Gremlin as one of the languages to interact with the Neo4j Server and embedding the Blueprints and Tinkerpop components as part of the main server distribution.
  • Bulbflow: Bulbs is a Python persistence framework for graph databases that connects to Rexster, and it enables you to run Gremlin queries against any Blueprints-enabled database. Bulbflow is a Python toolkit for building Web applications that is based on Bulbs and Flask.
  • AgentSpring: AgentSpring is an agent-based modeling and simulation framework that uses a graph to represent the simulated environment. It uses Gremlin to query the simulated world and mine it for patterns.
  • Social Networks in Archival Context: SNAC is an NEH funded research project this is trying to “unlock” the historical social network relationships that exist in descriptions of archival collections. Gremlin and rexster are used to power interactive graph visualizations. Gremlin code and graph database (released as graphML) are open source.
